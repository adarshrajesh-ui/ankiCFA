WHAT CHANGED  (config + type-annotation only; no runtime behavior changes)
------------------------------------------------------------------------------
1) .mypy.ini  mypy_path += `.` (repo root)
   Makes the genuine CFA package imports resolve explicitly on mypy's search path,
   e.g. `from cfa.ai.llm_client import complete` (qt/aqt/cfa_tab_fill.py,
   tools/cfa/f9_reachability.py). Mirrors the runtime PYTHONPATH the just recipes
   use (…:.). Purely additive; existing anki/aqt module naming is unchanged because
   mypy resolves each file against its nearest, more-specific base (pylib, qt, …).

2) .mypy.ini  targeted per-module ignore_missing_imports:
   - [mypy-openai] / [mypy-openai.*]  — OPTIONAL third-party SDK, declared only in
     cfa/ai/requirements.txt, not installed in the type-check venv, imported inside
     try/except ImportError (AI-off fallback). Matches the existing [mypy-bs4] idiom.
   - [mypy-ethics_scoring] [mypy-ai_grading] [mypy-passages] [mypy-import_pairs]
     [mypy-build_cfa_deck] [mypy-seed_collection]  — first-party CFA helper modules
     that are loaded by BARE name after a runtime `sys.path.insert(0, …)` (e.g.
     qt/aqt/cfa_seed.py inserts cfa/ethics_pairs; the tools/cfa proof scripts import
     a sibling from their own dir). mypy cannot follow runtime sys.path edits. These
     dirs were deliberately NOT added as mypy roots:
       * tools/cfa is already type-checked as the `tools.cfa.*` package, so adding it
         as a root triggers a fatal "Source file found twice under different module
         names" (verified empirically — see NOTES.md), and
       * rooting cfa/ethics_pairs pulls out-of-scope proof scripts into the checked
         graph (surfaced 5 pre-existing dict-typing errors in tools/cfa/render_f2_ai_grade.py,
         a file P4 does not own), i.e. it would WIDEN the target's designed folder scope.
     A targeted per-module ignore clears the import-not-found noise without blanket
     ignores and without changing the check:mypy target's scope. It does not hide any
     error that was previously being checked (these call sites were already `Any`).

3) tools/cfa/sync_roundtrip.py  (OWNED) — type-annotation only:
   - `_add_card(...) -> CardId`     (was `-> int`; find_cards returns CardId)
   - `_review(col, cid: CardId, ease: Literal[1, 2, 3, 4])`  (was `cid: int, ease: int`)
   - added imports `from typing import Literal` and `from anki.cards import CardId`
   With `from __future__ import annotations` in effect these annotations are never
   evaluated at runtime, so behavior is unchanged (CardId is a NewType == int).
