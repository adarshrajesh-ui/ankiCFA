WHY THE CONSOLE SCRIPT MATTERS (reproduction fidelity):
  `python -m pytest` implicitly prepends the current working directory
  (the repo root) to sys.path, which MASKS this bug — under `-m` the test
  passes even with the restricted PYTHONPATH. The real gate invokes the
  `pytest` console script (out/pyenv/bin/pytest), which does NOT add the
  CWD to sys.path. So the faithful repro below uses the console script,
  exactly like the gate. (`aqt`/`anki` resolve via the editable-install
  .pth for source dirs qt/ + pylib/; the generated modules resolve via the
  out/pylib + out/qt entries; only the repo-root package `cfa` has nowhere
  to resolve from -> ModuleNotFoundError.)
