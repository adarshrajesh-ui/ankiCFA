FINDING (MAJOR, accessibility)
------------------------------
The Deadline planner (/cfa-deadline/{deckId} -> CfaDeadlinePage.svelte) flagged
an at-risk row — a studied card whose predicted exam-day recall has fallen below
the 0.85 threshold — by colouring the recall figure warn-orange ($cfa-warn
#b45309) and NOTHING ELSE. `recallCell()` returned the identical "62.1%" string
for an at-risk row as it would for any figure; the ONLY difference between an
at-risk and a healthy studied row was the CSS `color`. A reader with a red-green
colour vision deficiency (protanopia / deuteranopia — ~8% of men, ~0.5% of
women) therefore has no reliable cue to which rows are at risk.

SCIENTIFIC GROUNDING (computed, not by-eye)
-------------------------------------------
Dichromacy was SIMULATED on the brand semantic hues (Viénot/Brettel-style
severity-1 matrices, applied in linear RGB) and the perceptual separation
measured as CIE76 ΔE in L*a*b*:

  pair            normal ΔE   deuteranopia      protanopia
  pass  vs warn      84.3      30.6  (36%)       15.2  (18%)   <- COLLAPSES
  warn  vs muted     76.0      68.7  (90%)       59.5  (78%)
  warn  vs ink       87.1      82.9  (95%)       70.3  (81%)   <- survives

The classic green/orange (pass/warn) hue pair collapses to a near-
indistinguishable band once red-green perception is removed (>65% of the
separation lost under protanopia) — the canonical reason a hue-only status flag
is unsafe. HONESTY NOTE: the specific warn(orange)-vs-ink(navy) pair used in the
Deadline table retains its LUMINANCE separation (81% under protanopia), so this
is a Level-A COMPLIANCE gap (colour used as the sole visual channel), not a
luminance collapse. The redundant non-colour cue is required regardless — Level
A is a binary conformance bar, and the product uses the full pass/warn/fail
triad elsewhere.

FIX (parity-safe, presentation-only)
------------------------------------
Add a redundant NON-colour cue to at-risk rows so the state survives with no
colour perception at all:
  * a SHAPE marker "▲" (a solid up-triangle = "attention" by shape) rendered
    before the recall figure — new pure helpers `isAtRisk()` / `riskMarker()` in
    ts/lib/cfa/pages/deadline.ts;
  * a visually-hidden screen-reader label "at risk:" (RISK_LABEL) via a
    `.cfa-deadline__sr` clip-rect class, so assistive tech announces the state
    too (WCAG 1.1.1 / 4.1.2).
New cards (no memory model) are never "at risk" and never carry the marker; the
warn colour is preserved for sighted users (defence in depth). The shared engine,
the deadline payload shape and every number are untouched.
