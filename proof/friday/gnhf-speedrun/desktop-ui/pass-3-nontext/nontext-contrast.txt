Phase B — Pass 3 (ruthless) — DESKTOP finding #2
WCAG 2.1 SC 1.4.11 Non-text Contrast: interactive control boundaries
====================================================================

Iteration 45. The Pass-3 text-contrast audit (iter 43) only checked text on
backgrounds. This finding extends the SAME scientific method (ratios computed
from the token hex values, no by-eye guessing) to NON-TEXT contrast — the
boundary that makes an interactive control perceivable.

FINDING (D-P3-2, MAJOR accessibility)
-------------------------------------
The CFA web design system draws the boundary of its interactive controls with
the DECORATIVE hairline token $cfa-line (#e7e9ec):
  - ts/lib/cfa/pages/CfaHomePage.svelte  .cfa-home__cta   (secondary Study CTAs)
  - ts/lib/cfa/pages/CfaHomePage.svelte  .cfa-home__chip  (AI / Browse-decks pills)
  - ts/lib/cfa/pages/CfaDeadlinePage.svelte .cfa-deadline__date-input (date field)

Each control has a WHITE fill (#ffffff) on the near-white page tint (#fafbfd),
so the hairline border is the ONLY thing that says "this is a control". But the
hairline measures only:
  $cfa-line #e7e9ec  vs white  #ffffff : 1.22:1   (FAIL — needs >=3:1)
  $cfa-line #e7e9ec  vs page   #fafbfd : 1.17:1   (FAIL)
  $cfa-line #e7e9ec  vs surface#f3f6f8 : 1.12:1   (FAIL)
WCAG 2.1 SC 1.4.11 requires the visual boundary of an active user interface
component to reach 3:1. At 1.2:1 the four secondary Study tiles and the two
footer chips effectively FLOAT — white-on-near-white — so a low-vision user
cannot tell they are tappable. (The primary CTA is exempt: it has an
accent-soft fill + accent border. Card edges, table rules and dividers are pure
decoration, which 1.4.11 explicitly exempts, so they keep the hairline.)

FIX (parity-safe — the iter-26/43 pattern: add a token, never change a
parity-gated value)
-------------------------------------------------------------------------------
New web-only token in ts/lib/cfa/_tokens.scss:
  $cfa-control-border: #7e8896;  // navy-tinted mid grey
Measured as a control edge:
  #7e8896 vs white  #ffffff : 3.59:1   (PASS)
  #7e8896 vs page   #fafbfd : 3.47:1   (PASS)
  #7e8896 vs surface#f3f6f8 : 3.31:1   (PASS)
and far lighter than $cfa-muted (#4d5c6d, 6.85:1 on white) so it reads as an
EDGE, never as text. Repointed the three interactive-control borders above to
$cfa-control-border; $cfa-line (parity-gated, byte-identical to cfa_style.py)
stays on every decorative hairline.
