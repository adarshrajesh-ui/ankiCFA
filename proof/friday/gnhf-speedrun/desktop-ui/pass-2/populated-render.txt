Phase B — DESKTOP Pass 2 (harsher): the POPULATED CFA render
=============================================================

Named must-fix (SPEEDRUN-PLAN §4): "the desktop Readiness screen ... make it
actually render the three scores, ranges, and coverage map with real data."

Pass 1 proved the honest zero-review ABSTAIN state (the true first-run screen).
This pass proves the OTHER half — the state a RETURNING learner sees, with a
real graded review history behind it: real score ranges + a fully lit-up
coverage map, NOT abstain. The numbers are computed by the same shared engine
(ComputeCfaScores) from a real seeded study history — honest evidence, not a
static mock or a fabricated score.

How it is produced (reproducible)
---------------------------------
  just cfa-capture-populated
    -> ./ninja pyenv ts:generated pylib qt
    -> CFA_SEED_REVIEWS=1 CFA_UI_OUT=.../desktop-ui/pass-2 playwright test
       cfa_readiness_populated

The launcher (qt/tests/launch_anki_for_e2e.py), when CFA_SEED_REVIEWS is set,
opens the profile's collection headlessly BEFORE mediasrv, seeds the CFA decks
(seed_collection) + a graded review history (tools/cfa/seed_reviews.py) that
crosses every give-up threshold, then closes it; mediasrv opens the populated
collection and the Svelte pages render real data. Default runs (no env var) keep
the fresh-profile abstain behaviour, so the Pass-1 abstain gate stays green.

Seeded history (deterministic)
------------------------------
  cards touched   : 80  (8 per topic x 10 canonical topics)
  graded reviews  : 320 (>= MIN_GRADED_REVIEWS 200)     across distinct days
  first exposures : 80  (>= MIN_FIRST_EXPOSURES 30)
  topic coverage  : 100% (10/10)  (>= MIN_TOPIC_COVERAGE 50%, no high-wt skipped)
  accuracy mix    : ~70% correct (ease 3) / ~30% incorrect (ease 1)
  FSRS state      : fixed stability 30d, per-card elapsed 0.3x..5x stability ->
                    a realistic recall SPREAD (not a fake flat 100%)
