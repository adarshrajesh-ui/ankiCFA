D8-1  (MAJOR)  stock-blue leak on deck names + "New" counts
------------------------------------------------------------------
Defect (before): although the iter-29/31 shell + toolbar work branded the top
bar and page tint navy, the deck LIST itself still leaked stock Anki blue in two
places, exactly mirroring the mobile M6-1 defect:
  * filtered/dynamic deck NAMES — "Ethics Pairs", "Study — Ethics Minimal-Pairs",
    "CFA Exam Priority" — rendered in stock blue `--fg-link` (#1d4ed8), because
    the base `.filtered { color: var(--fg-link) !important }` beat the CFA
    `a { color: ink }` rule (both specificity AND `!important`).
  * the "New" card COUNT numbers (20 / 29 / 99 / 1) rendered in stock blue
    `--state-new` (#3b82f6) via `.new-count { color: var(--state-new) }`.
Together the deck list — the first screen after Home — read blue-accented despite
the navy CFA shell: the exact "non-native-CFA feel" the objective flags.

Fix (`cfa_chrome._deckbrowser_css()`, presentation-only, no token value change):
  a.deck            { color: #122B46 !important; }   /* deck names → brand navy */
  a.deck.filtered   { color: #122B46 !important; }   /* beats .filtered !important */
  .new-count        { color: #122B46 !important; }   /* New count → brand navy */
The CFA study decks are curated study modes, not an Anki internal to surface in
loud blue, so their names read in brand navy → one cohesive branded deck list.
The "New" count keeps the three-way distinction (new = navy, learn = red,
review = green); the learned Anki learn/review count semantics are UNCHANGED
(matching the M5-2 / M6-1 decision), and the orange top-bar accent stays the
single warm accent.

Verified (after): all four deck names + all "New" counts render brand navy;
"Learn" stays red (the `2`), "Review" stays green (12 / 4 / 8); zero counts stay
faint grey. See desktop-ui/pass-2/d8-deck-browser.png.
